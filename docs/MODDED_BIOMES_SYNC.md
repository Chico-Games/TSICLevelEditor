# Modded Biomes & Biome Sync

This editor's biome list is **generated from the TSIC mod definitions**, not hand-maintained.
`com.chicogames.default` (the base game's exported definitions) is the single source of truth,
vendored into this repo and synced into `config/biomes.json`.

## Where biomes come from

- **`vendor/default-project/`** — a git **subtree** of
  [`Chico-Games/tsic-default-project`](https://github.com/Chico-Games/tsic-default-project)
  (branch `beta`). It holds the full default-mod export, including
  `biome_definitions/BD_*.json`. This is the source of truth.
- **`config/biomes.json`** — a **generated artifact**. Don't hand-edit the `Biome_*` tilesets,
  `enumMappings.ETileBiome`, or the biome `valueRange`; they're regenerated by the sync tool.
  (The non-biome blocks — Difficulty/Hazard/Height tilesets, layers, sizes — are preserved.)

Each biome's **color, display name, tag, and role** come from the mod export. Editor-only
presentation (icon, category, description) and the canonical palette **value** are preserved
across syncs for stability; brand-new modded biomes get appended indices + role-based defaults.

## Updating the vendored definitions

```bash
git subtree pull --prefix=vendor/default-project default-project beta --squash
node tools/sync-biomes.mjs            # regenerate config/biomes.json
```

(The `default-project` remote is `https://github.com/Chico-Games/tsic-default-project.git`.)

## The sync tool

```bash
node tools/sync-biomes.mjs [--project <dir>] [--mods <dir> ...] [--out <file>] [--check]
```

| Flag | Meaning |
|---|---|
| `--project <dir>` | Default mod export dir (default `vendor/default-project`). |
| `--mods <dirs>` | Add-on mod dirs to overlay, last-writer-wins by biome tag. New biomes get appended palette indices. |
| `--out <file>` | Config to rewrite (default `config/biomes.json`). |
| `--check` | Don't write; exit 1 if the config is out of date (use in CI / a `post-merge` hook). |

Example — bake the default mod **plus** an add-on mod into the editor:

```bash
node tools/sync-biomes.mjs --mods ../some-mod/com.example.forest
```

## Map validation against the active mod set

On import, `js/dynamic-validator.js` (via `js/mod-catalog.js`) checks that a map only uses
biomes defined by the active mod set baked into `config/biomes.json`:

- a biome **value** painted into a layer that isn't a defined biome → **error**
- a `color_mappings.biomes` entry whose **tag** isn't in the active set → **error**
- a `color_mappings` palette index that disagrees with the active set → **warning**

`config.biomeSource.syncedFrom` records which mods the current config was built from, and each
`Biome_*` tileset carries a `mod` field for provenance.

## Deployment note

`vendor/default-project` (~88 MB) is committed for the source-of-truth subtree but is **excluded
from the published site** by `.github/workflows/deploy-pages.yml` — the live site only ships the
small generated `config/biomes.json`. To use it, set repo **Settings → Pages → Source = GitHub
Actions**.
